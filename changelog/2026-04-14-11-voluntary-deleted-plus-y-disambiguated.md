## Change

Deleted the `voluntary` bench reason label from `rules/02-bench-rotation.md` → "Bench reason vocabulary". Removed the incidental `voluntary` mention from the *Explicit user exemption* paragraph of *Raid leader's discretionary bench picks*. Clarified in `reference/file-operations-manual.md` → Event: New signup screenshot received → Step 2 that Discord's `+Y` header count is a conjoined aggregate of three distinct per-person states — *bench signup*, *tentative/TBC*, and *late* — that must always be disambiguated per name. Documented how each state feeds (or doesn't feed) the set file and `bench-history.md`: bench signups flow into `## Bench` as `leader choice`, tentatives stay out of roster decisions until resolved, lates are roster members arriving after start. Added a `**Tentative ({N}):**` sub-line to both set templates (`reference/templates/25man-set.md`, `reference/templates/karazhan-set.md`) so TBC signups have a canonical home.

## Reason

The `voluntary` label and the "bench = player self-benches" framing were both wrong: players don't bench themselves in this project's model, the raid leader (and the rules) enforce benches. The label added no information that `leader choice` didn't already capture, and it invited misreadings of the `+Y` header count as if that count were itself a bench category.
