# 2026-04-13 — Default priority for new players

## Change
Added a sentence to `rules/04-players.md` → "Raid spot priority" stating that the default priority for a newly-added player is `2`. Priority 1 (always plays) and priority 3 (last resort) require explicit user instruction.

## Reason
The rule file previously didn't say what priority to assign when a player who isn't in the roster appears for the first time. A fresh Claude session would have had to guess.
